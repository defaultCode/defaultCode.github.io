这次, 我将Singleton类中的private static Singleton singleton = new Singleton();移动到了public static int count1;之前, 而结果发生了变化.
为什么?
回顾之前所学的, 类的加载中有个连接阶段, 其中还分了三个阶段, 这三个阶段中有一个准备阶段, (该阶段为类的静态变量分配内存, 并将其初始化为默认值).
而初始化阶段是为类的静态变量赋予正确的初始值.
这里我们可以看到Test的main方法中,我们调用了类的静态方法. 所以毫无疑问, Singleton进行了初始化.
知道了这些, 我们连贯地将这个过程理一下:

  • Singleton类进入了准备阶段, 为所有的静态变量进行赋值. 也就是该阶段count1,count2均为0, 而singleton为null
  • Test调用了getInstance方法, 导致``Singleton`进入了初始化阶段.
  • 初始化阶段会为类的静态变量赋予正确的初始值…会将Singleton代码顺序执行进行赋值操作.
  • 则会先进行private static Singleton singleton = new Singleton();的new对象操作, 然后执行构造方法中的count1++;count2++;, 此时count1和count2的值均为1.
  • 再向下执行, public static int count1;public static int count2 = 0; 因为public static int count1没有赋值操作, 所以该句没有进行任何操作, 而public static int count2 = 0;被执行, 所以此时count2被改变为了0.
