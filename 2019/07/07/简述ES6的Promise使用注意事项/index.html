前言

最近需要对小程序的请求方式进行封装, 遂学习了ES6中Promise的用法, 本以为学一个ES6的新语法而已嘛, 用得了多长时间呢? 却万万没想到, 还是挺难的, 其底层实在是还没搞明白.

由于网上其实有很多误导性的文章也对学习造成一定的阻碍, 所以在这里把我学习到的内容记录下来.

感性认识Promise

在学习之前我们先了解一下什么是Promise, 我们为什么需要学习它?

Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。—摘自阮一峰《ECMAScript6入门》

这个解释也还算比较容易理解, 就是异步编程的解决方案. 比传统的回调函数合理和强大, 也就意味着可以取代传统的回调.

传统回调问题又有什么不好的? (回调地狱)

有时候我们一个请求是依赖另一个请求的结果的.
举个例子, 比如面试的过程…

  1. 我投递简历, 申请面试机会(发起第一次请求…)
  2. 人事筛选了面试者, 电话通知我的简历通过了希望我能够面试(服务器响应), 于是我带着简历过去相应公司面试(第二次请求)
  3. 前台验证了身份后, 安排我写笔试题(服务器响应). 于是我写完笔试题后, 交还到人事手上(第三次请求)
  4. 笔试题通过后, 人事通知我可以与技术总监面试(服务器响应). 于是我与技术总监面谈(第四次请求)
  5. 技术总监通过了, 告诉我可以跟人事再谈后续的内容(服务器响应), 然后我再找人事面谈(第五次请求)
    …

大家应该不难看出, 这些步骤是顺序执行的, 比如我必须简历通过了才有机会到公司面试, 前台必须验证了身份(学历等相关信息), 我才有资格拿到笔试题开始做题…
这样子的请求就是我说的有时候我们一个请求是依赖另一个请求的结果的.

Promise的基本概念

废话说了这么多, 我们开始正式的学习吧!
在上述的展示中, 其实Promise的基本长相大家还没有看到, 这里给大家看一下:

我们看到了Promise的基本长相, 但对它的用法还不了解, 下面我们一步步学习.
Promise对象对象代表一个未完成、但预计将来会完成的操作. 它有三种状态:

  • pending: 初始值, 不是fulfilled, 也不是rejected
  • fulfilled: 代表操作成功
  • rejected: 代表操作失败

Promise有两种状态改变的方式, 既可以从pending转变为fulfilled, 也可以从pending转变为rejected. 一旦状态改变, 就「凝固」了, 会一直保持这个状态, 不会再发生变化. 当状态发生变化, Promise.then绑定的函数就会被调用.
太官方了? 用我的话给大家解释一下上述Promise样例的代码:

  1. 先创建了Promise对象, 构造函数中传入一个函数(后续将这个函数成为func), 携带两个回调函数resolve和reject.
  2. 在func中执行发请求的操作, 如果请求成功了, 调用resolve, 请求失败就调用reject (很多教程中比较喜欢将这个调用过程称之为改变状态, 状态改变了自动调用). 由于这些数据是在请求后得到的, 所以请求到了结果后, 这两个回调函数就会根据结果调用. 达到我们想要的结果.
  3. 在then中将resolve和reject的具体函数逻辑指定. 调用时将携带的参数data进行处理.
  4. catch可以捕获在Promise中出现的错误.

基本用法便是如此了.

接下来说一下使用Promise必须注意的事项

注意事项

Promise 会吃掉错误!

如果在Promise代码中出现了错误, 但没有使用catch进行捕获, Promise对象抛出的错误不会传递到外层代码, 即不会有任何反应. 也就是出了错你不能够发现, 这是相当恐怖的, 希望大家注意, 只要记得每一个Promise都使用catch进行异常捕获, 就不会出问题啦!
