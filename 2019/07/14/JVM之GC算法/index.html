JVM之GC算法
发表于 2019-07-14| 更新于 2019-10-14|Java
字数总计: 1.7k|阅读时长: 5 分钟|阅读量:

前言

由于对JVM有一定的了解, 总能听到类似新生代, 老年代, 永久代这些描述堆内存区域结构的词汇, 但总无法理解这些区域的作用, 和GC的关联之处. 所以打算这段时间将JVM体系知识深入了解, 首先就将这些之前的模糊概念搞明白.

本文相关内容基于JDK8进行展开, 由于JDK8版本中不再存在永久代区域, 不再对其进行阐述.

一些常用GC算法

以前看不明白这些(堆内存区域)概念的主要原因在于不了解GC算法的时候就看这些概念, 无法结合应用场景, 所以总是无法理解, 所以我们先不介绍新生代和老年代的概念, 先了解一下GC算法.
概念就不过多介绍了, GC算法就是JVM进行GC时的算法. 实在是没啥好解释的…

常用的GC算法:

  1. 引用计数法(Java中没有使用这个算法)
  2. 可达性分析法
  3. 标记清除法
  4. 标记压缩法
  5. 复制算法
  6. 分代收集法

其实这些算法都比较易懂, 接下来我会对这些算法尽可能简单地进行一一介绍.

GC Roots
接下来可能会多次提到GC Roots, 所以先说一下, GC Roots的对象在java中包括以下几种:

  1. 虚拟机栈（栈帧中的本地变量表）中引用的对象.
  2. 方法区中的类静态属性引用的对象.
  3. 方法区中的常量引用的对象
  4. 本地方法栈中JNI（Native方法）的引用对象

接下来我会用根对象来表示GC Roots.

引用计数法

引用计数法, 也就是将一个对象的引用做用的个数记录下来, 比如有一个引用就记为1, 两个引用就记为2, 以此类推… 则如果引用数为0的话就可以将其回收了. (如果有循环引用则就算根对象(GC Roots)对其没有引用也不会回收, 这是一个极大的弊端, 所以java中没有使用这个算法)

可达性分析法

这个算法的基本思路就是从根对象作为起始点进行遍历, 搜索所走过的路径称为引用链, 当一个对象到根对象没有任何引用链相连时, 就证明此对象是不可用的. 便可以进行回收.
java中是通过可达性分析来进行判断对象是否存活.
在书中看到可达性分析中的特殊现象做了个记录,感兴趣的朋友可以看看>>对象已死吗?

标记清除法

简单来说, 这个算法有两个步骤: 标记, 清除.

  1. 标记阶段: 找到所有可达(存活)对象, 进行标记.
  2. 清除阶段: 遍历堆, 把未标记的对象及进行回收.

缺点:

  1. 由于涉及大量遍历工作, 执行性能比较低, 会导致”stop the world”时间较长.
  2. 对象被清除后, 被清理的对象留下的内存空缺会造成内存不连续, 浪费空间.
标记压缩(整理)法

标记压缩法在标记清除法的基础上做了一些优化, 适用于存活对象比较多的场景.
也比较容易理解, 同样是两个步骤: 标记, 整理.

  1. 标记阶段: 找到所有可达对象, 进行标记.(与标记清除法一样)
  2. 整理阶段: 移动所有存活的对象, 按内存地址有序排列, 然后将末端内存地址以后的内存全部回收.


可以直观地感受到, 这个算法弥补了标记清除法中内存区域分散的缺点.

缺点:
效率比较低, 需要标记对象还要整理所有存活对象的地址.

复制算法

复制算法简单来说就是把内存分为两个部分, 在垃圾回收时, 将正在使用的存活对象赋值到一份空白的内存中, 将正在使用的内存空间所有对象清除.

优缺点比较明显.
优点:
复制算法只需要将存活对象的指针修改, 按顺序分配内存, 非常高效.
缺点:
因为需要将内存一分为二, 所以空间上一直需要保留一般的未使用, 这样非常消耗内存.

分代回收算法

理论上这个算法不算一般的GC算法, 因为这只是java中的实现的方式, 其实现在JVM使用的并不是只使用上述GC算法中的一种，而是分代收集的算法. 这也是本文的重点.

GC的回收机制其实不仅仅只回收堆中的对象, 但这里我只对堆进行详述.

分代回收算法就是将内存根据对象存活的周期划分为几块. 一般是把堆分为新生代和老年代.
短命对象存放在新生代中, 长命对象放在老年代中.

其中新生代中存在三个区域, 一个是新创建的对象放置的区域, 被称为”Eden”, 还有两个被称为”survivor”的区域.

这些区域的作用?
看图来理解就很容易明白.

  1. 对象新创建时, 会将对象存于新生代中的Eden区域, 进行GC时, 会将大对象优先复制到老年代(由于对象比较大, survivor区域空间有限, 特殊处理).
  2. 将存活的对象复制到一个空白的survivor中.
  3. 将正在使用的survivor中存活的对象复制到空白的survivor区中.
  4. 将处理过的Eden与survivor中的所有内容清空.
  5. 老年代以标记清除法进行回收.完成整个GC(不是每次GC都进行老年代回收).

注: 过程中每次GC都会把存活的对象年龄加1, 一旦survivor区域中有GC多次但没有被回收的对象, 则将其复制到老年代中.

可以通过配置-XX:MaxTenuringThreshold=n来指定对象经历多少次GC后移至老年代中. 默认值为15(次).
